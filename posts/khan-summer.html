I spent the summer in California, at the Khan Academy. This is some of what I did there.

Projects

Accurate Video Log Recording

I spent my first day improving the way we record how much of a video users have watched. We used to poll every ten seconds to see where in the video the user was and report back to the server if they had watched a certain amount. I changed the code to use the YouTube api, which allows us to record every time the user pauses or skips in the video. The idea is to have the exact segments of video that were watched so we can analyze that data and see if people are getting bored or going back to review certain parts of videos. This was a great project to spend my first day on.

Phantom Users

Next I teamed up with another intern, Parker, on the phantom users project, which was not as mysterious or scary as it sounds. Before we added our magic, users had to create an account on the site to use exercises. People have always been able to access the videos and we wanted them to be able to try out the exercises as well, so we could prove to them that it's worth it to create an account. So unfortunately "phantom users" has nothing to do with the physical manifestation of the soul or spirit of a deceased person, it could just have easily been called "unregistered users" (if we were lame). This project went well and gave us the opportunity to dig into the guts of the site.

Video Progress Indicators

I wrote about this in more detail last time. Briefly, I added icons to video links around the site to indicate if the video has been started or finished.

Deploy Script Improvements

In the middle of the summer we were going through a bit of a performance crisis. Every day we would see latency spikes where users would have to wait far too long for pages to load. The big fix was some work done by Desmond to switch to the High Replication Datastore but in the meantime I wanted to do something to help. The idea, suggested by Ben Alpert, was to automatically include images in our stylesheets using data-uris. This reduces the number of requests the browser must make, hopefully speeding up the page. I knew my idea probably wouldn't get approved if I asked permission to do it, so I did it first. If you're using a browser better than IE 7, you see this system in work any time you visit the site (it looks exactly the same, which is good).

To update the live version of the site you must run a deploy script that used to take about ten minutes, which is annoying and unproductive to wait for. So, at the same time as the data-uri project I was working on a system to speed up the deploy script. The cause of the slowness was a horrible css compressor that will not be named, which could take several minutes to compress a large stylesheet. What I did was cache the stylesheets on the deployer's computer so they don't need to be run through the compressor again if they haven't changed since the last time. I consider this project a bit of a failure, since we recently replaced the css compressor which sped things up enough that this work is not necessary anymore and should probably be removed from the code. Lessons learned: challenge assumptions and be lazy.
